# 用户分配记录查询工作台 - 后端接口设计

## 1. 后端职责

后端只做三件事：

1. 接收前端传入的查询条件。
2. 根据渠道生成 BPO / TMK 查询 SQL。
3. 返回统一结构给前端展示和导出。

前端不直接接触 SQL，也不区分底层表结构。

## 2. 查询接口

### 接口地址

`POST /api/allocation/query`

### 请求参数

```json
{
  "uid": "123456789",
  "channel": "all",
  "dateMode": "specific",
  "date": "2026-05-18"
}
```

### 参数说明

| 参数 | 类型 | 必填 | 说明 |
|---|---|---|---|
| uid | string | 是 | 用户 UID，只支持单个 UID 查询 |
| channel | string | 是 | `all` / `bpo` / `tmk` |
| dateMode | string | 是 | `specific` 指定日期；`all_time` 不限制时间 |
| date | string | 条件必填 | `dateMode=specific` 时必填，格式 `YYYY-MM-DD` |

### 返回参数

```json
{
  "uid": "123456789",
  "hasAllocation": true,
  "latestDt": "2026-05-18",
  "summary": {
    "bpoCount": 1,
    "tmkCount": 2
  },
  "bpoRecords": [
    {
      "dt": "2026-05-18",
      "userid": "123456789",
      "phone": "138****5678",
      "leadType": "externalLead",
      "userType": "pediaStock",
      "grade": "unset",
      "rank": 28,
      "extraInfo": "unset"
    }
  ],
  "tmkRecords": [
    {
      "dt": "2026-05-18",
      "user_id": "123456789",
      "lead_channel": "tmk_hunt",
      "hunt_lead_type": "stock_user",
      "grade": "S2",
      "queue_rnk": "15"
    }
  ]
}
```

## 3. 导出接口

### 接口地址

`POST /api/allocation/export`

### 使用场景

支持两种导出：

1. 导出某个 UID 的明细。
2. 导出指定日期下的全部明细。

### 请求参数

```json
{
  "uid": "",
  "channel": "all",
  "dateMode": "specific",
  "date": "2026-05-18"
}
```

### 导出限制

- 如果 `uid` 为空，必须选择 `dateMode=specific`。
- 不允许在 `uid` 为空时导出 `all_time`，避免导出数据过大。
- 导出格式建议为 CSV。

### CSV 字段

| 字段 | 说明 |
|---|---|
| 渠道 | BPO / TMK |
| 分配日期 | dt |
| 用户UID | user_id / userid |
| 手机号 | BPO 有，TMK 可为空 |
| 线索类型 | leadType / hunt_lead_type |
| 用户类型/线索渠道 | userType / lead_channel |
| 年级 | grade |
| 排名 | rank / queue_rnk |
| 扩展信息 | extraInfo |

## 4. BPO SQL 模板

### 指定 UID 查询

```sql
select
    coalesce(default.encrypt_with_passid(call_user_type,'00stuvwx7a'), '0') as phone,
    user_id as userid,
    'externalLead' as leadType,
    dt,
    'unset' as grade,
    case when channel='百科存量' then 'pediaStock' else channel end as userType,
    row_number() over(order by rand(2026)) as rank,
    'unset' as extraInfo
from dw_conan_ads.ads_eng_tmk_hunt_side_user_detail_di_no_sensitive_view
where call_user_type != '0'
  and user_id = '${uid}'
  -- 指定日期时加：and dt = '${date}'
```

### 指定日期导出全部

```sql
select
    coalesce(default.encrypt_with_passid(call_user_type,'00stuvwx7a'), '0') as phone,
    user_id as userid,
    'externalLead' as leadType,
    dt,
    'unset' as grade,
    case when channel='百科存量' then 'pediaStock' else channel end as userType,
    row_number() over(order by rand(2026)) as rank,
    'unset' as extraInfo
from dw_conan_ads.ads_eng_tmk_hunt_side_user_detail_di_no_sensitive_view
where call_user_type != '0'
  and dt = '${date}'
```

## 5. TMK SQL 模板

### 指定 UID 查询

```sql
SELECT
    dt,
    user_id,
    lead_channel,
    hunt_lead_type,
    grade,
    queue_rnk
FROM dw_ads.ads_eng_tmk_hunt_all_user_detail_with_grade_di_no_sensitive_view
WHERE user_id = '${uid}'
  -- 指定日期时加：and dt = '${date}'
ORDER BY
    dt DESC,
    CAST(queue_rnk AS BIGINT)
LIMIT 100000
```

### 指定日期导出全部

```sql
SELECT
    dt,
    user_id,
    lead_channel,
    hunt_lead_type,
    grade,
    queue_rnk
FROM dw_ads.ads_eng_tmk_hunt_all_user_detail_with_grade_di_no_sensitive_view
WHERE dt = '${date}'
ORDER BY
    CAST(queue_rnk AS BIGINT)
LIMIT 100000
```

## 6. 参数校验

后端需要校验：

- `uid` 查询场景：UID 不能为空，且建议只允许数字。
- 指定日期场景：`date` 不能为空，格式必须为 `YYYY-MM-DD`。
- 导出场景：如果 UID 为空，只允许导出指定日期。
- `channel` 只能是 `all` / `bpo` / `tmk`。
- SQL 参数必须走参数化或白名单拼接，避免 SQL 注入。

## 7. 第一版实现建议

优先顺序：

1. 先实现 `POST /api/allocation/query`，支持指定 UID + 指定日期。
2. 再支持指定 UID + 不限制时间。
3. 再实现 `POST /api/allocation/export`，支持指定日期导出全部。
4. 最后接入权限控制和云端部署。

第一版不建议开放“不填 UID + 不限制时间”的查询和导出。
